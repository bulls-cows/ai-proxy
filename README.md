# retry-model-request

一个给 AI 编程助手用的代理工具。当 AI 调用模型接口失败时自动重试，让你的编程任务不会因为网络问题而中断。

## 为什么需要这个工具？

如果你用过 AI 编程助手（比如 CodeBuddy、Cursor 等），可能遇到过这种情况：

```
❌ 调用模型失败，错误码 429（请求太频繁）
❌ 任务被迫中断，刚才写的代码白费了
❌ 只能重新开始，浪费时间和 token
```

这些错误通常是暂时的，重试一两次就能成功。但 AI 助手不会自动重试，任务就这样中断了。

这个工具就是在 AI 助手和模型接口之间加一层代理，遇到错误自动重试，让你的任务顺利完成。

## 使用方法

### 第一步：安装

```bash
npm install
```

### 第二步：配置

在项目根目录创建 `.env` 文件，填入你要代理的接口地址：

```env
LOCAL_PORT=3000
TARGET_BASE_URL=https://你的模型接口地址
MAX_RETRIES=3
DELAY_MS=1000
RETRY_STATUS_CODES=429,500,502,503,504
```

**配置说明：**

| 配置项               | 是什么           | 怎么填                              |
| -------------------- | ---------------- | ----------------------------------- |
| `LOCAL_PORT`         | 代理服务的端口   | 随便填一个没被占用的端口，比如 3000 |
| `TARGET_BASE_URL`    | 要代理的目标接口 | 填你原来直接调用的那个地址          |
| `MAX_RETRIES`        | 失败后重试几次   | 建议填 3，重试太多次会等很久        |
| `DELAY_MS`           | 每次重试等多久   | 建议填 1000（1秒）                  |
| `RETRY_STATUS_CODES` | 哪些错误要重试   | 默认就行，不用改                    |

### 第三步：启动

```bash
npm start
```

看到这个就说明成功了：

```
🚀 TypeScript 本地代理服务已启动
👉 本地地址：http://localhost:3000
🎯 目标接口：https://你的模型接口地址
🔁 最大重试：3 次
```

### 第四步：让 AI 助手用这个代理

在 AI 编程助手的设置里，把模型接口地址改成代理地址：

```
原来：https://api.xxx.com/v1
改成：http://localhost:3000/v1
```

就这么简单！现在 AI 助手调用模型时，如果遇到错误会自动重试，不会再中断你的任务了。

## 常见问题

### 我不懂技术，能教我怎么用吗？

可以，按这几步来：

1. 下载这个项目
2. 打开终端，进入项目目录
3. 输入 `npm install` 回车，等它装完
4. 创建一个叫 `.env` 的文件，内容照抄上面的例子，把 `TARGET_BASE_URL` 改成你的接口地址
5. 输入 `npm start` 回车
6. 去你的 AI 编程助手设置里，把接口地址改成 `http://localhost:3000`

### 我怎么知道要代理什么地址？

看你的 AI 编程助手设置页面，里面会有一个「模型接口地址」或「API Base URL」之类的配置项，那个就是要代理的地址。

### 代理后速度会变慢吗？

正常情况下不会。只有遇到错误重试时才会多花时间，但总比任务中断强。

### 如果重试 3 次还是失败怎么办？

那说明问题不是暂时的，可能是：

- 接口地址填错了
- API Key 过期了
- 账户余额不足

这时候需要检查你的接口配置。

### 支持哪些 AI 编程助手？

理论上都支持，只要能在设置里修改接口地址就行。比如 CodeBuddy、Cursor、Continue 等。

## 开发模式

如果你要修改代码，可以用开发模式，改完自动重启：

```bash
npm run dev
```
